### Based on test 3.2, does a class that will create an array of objects always have to have both attributes and methods? Explain!

**Answer:**

No, a class does not always have to contain both attributes and methods.

However, in object-oriented programming, a class is usually designed to represent data (attributes) and behavior (methods).

Attributes store the object's data, while methods define what the object can do.

If a class only has attributes without methods, it only stores data.
If it only has methods without attributes, it may perform operations but does not represent specific object data.

For better design and encapsulation, a class should normally have both.

### What does the following program code do?

(Referring to creating an array like this:)

```java
Student[] arrayOfStudents = new Student[3];
```

**Answer:**

This code creates an array that can store 3 objects of type `Student`.

However, it only creates the array container, not the actual `Student` objects.

Each element inside the array is still `null` until we instantiate each object using:

```java
arrayOfStudents[i] = new Student();
```

### Does the Student class have a constructor? If not, why is the constructor called?

**Answer:**

Even if we do not explicitly write a constructor, Java automatically provides a default constructor.

This default constructor is generated by the compiler as long as we do not define any constructor ourselves.

That is why we can still call:

```java
new Student();
```

If we define a parameterized constructor and do not define a default constructor, then `new Student()` will cause an error.

### Why are the Student and StudentDemo classes separated?

**Answer:**

They are separated because each class has a different responsibility.

* `Student` class → represents the object (data model).
* `StudentDemo` class → contains the `main()` method to run and test the program.

This separation follows the principle of **separation of concerns**, making the program more organized, readable, and maintainable.


**Answer:**

```java
myArrayOfStudents[0] = new Student();
```

Then we can access its attributes.

# PRACTICUM – Parameterized Constructor

---

### Can a class have more than one constructor?

**Answer:**

Yes, a class can have more than one constructor. This is called **constructor overloading**.

Example:

```java
public class Student {
    String name;

    // Default constructor
    public Student() {
    }

    // Parameterized constructor
    public Student(String name) {
        this.name = name;
    }
}
```

Java differentiates constructors based on parameter type and number.

### 1️Why use FOR loop to create array of objects?

**Answer:**

A `for` loop is used because we know the number of elements (array length).

It allows us to systematically instantiate each object inside the array.


### Why use FOREACH loop to display data?

**Answer:**

A `foreach` loop is simpler and more readable when we only need to access each element without using its index.

Example:

```java
for (Dosen d : arrayOfDosen) {
    d.printInfo();
}
```

It makes the code cleaner and safer.
